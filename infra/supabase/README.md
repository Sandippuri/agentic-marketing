# Supabase infra

Three SQL files; apply in order on a new Supabase project.

1. **Migrations** — generated by `pnpm db:generate` into `packages/db/drizzle/`. Apply with `pnpm db:migrate` (or `pnpm db:push` while iterating in dev).
2. **`policies.sql`** — RLS + the publish-gate trigger. Run via Supabase SQL editor or `psql` after migrations land.
3. **`seed.sql`** — default settings rows.
4. **`views.sql`** — analyst rollups. Re-run any time the schema underneath changes.

The publish-gate trigger in `policies.sql` is the load-bearing safety check (plan §3 / §9). Don't drop it.

## Memory directories

`apps/manager/memory/` and `apps/distributor/memory/` are git-tracked Markdown. They are NOT in Supabase. Do not commit secrets there — `.gitignore` has the standard patterns but treat anything under `memory/` as if it could be public.
